### About

I specialize in finding the gap between what software is supposed to do and what it actually does — across manual test design, automated suites, API validation, and backend data checks. I enjoy building test coverage that scales with a codebase rather than chasing it, and I approach quality as a property the whole team owns, not a checkpoint at the end of a sprint.

My background is in enterprise GRC tools and web applications. I work in Agile teams as a full participant — sprint planning, defect triage, release sign-off — and I'm currently extending my skill set from test execution toward CI/CD-integrated automation.
